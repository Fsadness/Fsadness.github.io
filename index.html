格式化路径

实现一个对输入的路径进行处理、并输出正确结果的程序。具体要求如下：

  • 所有输入中的\被当做/加以处理，且在输出中被转换为/。
  • 连续多个/被压缩成一个。
  • 输入中单独的.被直接删除。若为连续的.，则参考下面一条规则。
  • 输入中的..会导致..之前的一级目录被删去。然而如果输入中已经不存在父目录，则直接输出Value Error。保证不会出现...、....等不合法的情况。
  • 输入中保证不出现字母加.等情况。
  • 如果输入为空，或者所有父目录被删去，则根据下面的规则输出空串或/。
    • 当且仅当输入以/开始（或以\开始）时，输出才以/开始。
    • 无论输入是否以/结尾，输出的结尾不带有/，除非输出是/。

使用之前学到的白盒测试和黑盒测试方法，尽量减少错误次数。能一次通过最好啦！

你可以使用任意语言完成这道题目，点击代码编辑区右上角的下拉选项就可以选择使用的语言了。

输入格式

输入$T(1 \leq T \leq 10)$行，每行为一个长度不超过$100$的字符串（包含字母、数字、.、\和/），表示输入的路径。

输出格式

输出$T$行字符串，表示处理后的$T$个路径。

gdb调试

需要在编译的时候使用-g开启调试选项

功能
  • 启动你的程序，可以按照你的自定义的要求运行程序
  • 可让被调试的程序在你所指定的调置的断点处停住。（断点可以是条件表达式）
  • 当程序被停住时，可以检查此时你的程序中所发生的事
  • 动态的改变你程序的执行环境
常用参数

continue c 继续程序的运行,直到遇到下一个断点
set var name=v 设置变量的值
start st 开始执行程序,在main函数的第一条语句前面停下来
file 装入需要调试的程序
kill k 终止正在调试的程序
watch 监视变量值的变化
backtrace bt 查看函数调用信息(堆栈)
frame f 查看栈帧
quit q 退出GDB环境

代码风格

防御式编程

防止程序使用者的部分恶意输入导致程序崩溃

  1. 别让代码附带有任何假设
  2. 没有被写下来的假设会导致各种问题
  3. 特别是代码规模变大时更是如此
检查
  1. 检查所有外来数据的值
  2. 检查所有输入的参数
  3. 决定如何对不符合预期的输入进行处理
  • 常见的检查项
    • 检查确认所有的数组没有越界
    • 解指针引用前确认他们都是非空的
    • 确保函数的参数都是有效的
    • 检查函数的返回值是符合预期的要求的
    • ……
  • 常见的处理方法
    • 使用一个无害的值
    • 使用最近一次的合法值
    • 记录一个警告的日志
    • 过程中抛出一个异常
    • 进入一个专门的异常处理子程序
    • 终止程序运行
    • ……

Git实战

通过这一章的学习，想必你已经掌握了 Git 的许多常用操作。接下来，你将要解决几个用 Git 操作完成的小任务。如果你可以完成这些任务，就标志着你真正地掌握了Git 的常用操作。

任务说明

任务一

你现在应该能在目录中看到两个文本文件：test.txt和input.txt。现在，将test.txt加入.gitignore中，并将对.gitignore的修改提交到master分支（可以写上任意 commit message）。

任务二

本地仓库里有一个分支back_branch，将这个分支的最近一次提交删除。

任务三

从master建立一个分支commit_branch，将input.txt文件的第三行修改为Hello Jisuanke!，把这次修改合并到最近的一次提交中，并修改 commit message 为'merged message'。

任务四

将分支conflict_branch分支合并到master，发生冲突时均以conflict_branch为准，之后提交修改（可以写上任意 commit message）并删除conflict_branch分支。

判分说明

题目共计$100$分，每个任务$25$分。只有当某个任务完全正确时，才能得到该任务的分数。

本地测试

在project目录下执行bash getgrade.sh，即可在终端内看到测试的分数。

提示

在终端里编辑 commit message （默认使用 Vim 编辑）的方法如下：

  1. 输入:1，跳转到第一行
  2. 输入i，开启编辑模式
  3. 编辑 commit message
  4. 按Esc退出编辑模式
  5. 输入:wq保存退出

你可以拖动终端区域的上沿，以让这个区域的面积更大。

组合命令

使用|分隔前后两个命令

git(进阶)

使用git分支与版本切换

git branch

从当前版本链中分支出一个新的版本链

git checkout

从一个分支切换到另一个分支（在本地中其他分支的文件存储于.git文件夹中）

组合命令： 创建并切换分支git checkout -b

​ 删除分支git branch -d

git stash

在切换分支前，将未提交的修改进行暂存

恢复原现场状态： git stash pop

git merge

将两个分支进行合并

在分支master上的时候，进行git merge develop是将develop合并到master上
（仅修改master）

  • master上没有新的提交

    • fast-forward合并 待合并分支领先目标分支的合并
    • non-fast-forward合并 将master与develop的最后一次提交进行选择式的合并，并在master上产生一个提交

  • master与develop上均有新的提交

    • 不可以进行fast-forward合并，只能进行non-fast-forward合并

    • 合并不冲突修改 默认进行

    • 有冲突修改 需要进行手动修改
