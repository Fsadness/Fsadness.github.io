Git实战

通过这一章的学习，想必你已经掌握了 Git 的许多常用操作。接下来，你将要解决几个用 Git 操作完成的小任务。如果你可以完成这些任务，就标志着你真正地掌握了Git 的常用操作。

任务说明

任务一

你现在应该能在目录中看到两个文本文件：test.txt和input.txt。现在，将test.txt加入.gitignore中，并将对.gitignore的修改提交到master分支（可以写上任意 commit message）。

任务二

本地仓库里有一个分支back_branch，将这个分支的最近一次提交删除。

任务三

从master建立一个分支commit_branch，将input.txt文件的第三行修改为Hello Jisuanke!，把这次修改合并到最近的一次提交中，并修改 commit message 为'merged message'。

任务四

将分支conflict_branch分支合并到master，发生冲突时均以conflict_branch为准，之后提交修改（可以写上任意 commit message）并删除conflict_branch分支。

判分说明

题目共计$100$分，每个任务$25$分。只有当某个任务完全正确时，才能得到该任务的分数。

本地测试

在project目录下执行bash getgrade.sh，即可在终端内看到测试的分数。

提示

在终端里编辑 commit message （默认使用 Vim 编辑）的方法如下：

  1. 输入:1，跳转到第一行
  2. 输入i，开启编辑模式
  3. 编辑 commit message
  4. 按Esc退出编辑模式
  5. 输入:wq保存退出

你可以拖动终端区域的上沿，以让这个区域的面积更大。
