gcc -D编译选项

宏名(每个宏名前后双下划线) 类型 意义

gdb调试

需要在编译的时候使用-g开启调试选项

功能
  • 启动你的程序，可以按照你的自定义的要求运行程序
  • 可让被调试的程序在你所指定的调置的断点处停住。（断点可以是条件表达式）
  • 当程序被停住时，可以检查此时你的程序中所发生的事
  • 动态的改变你程序的执行环境
常用参数
命令 命令 缩写 命令说明
list l 显示多行源代码
break b 设置断点,程序运行到断点的位置会停下来
info i 描述程序的状态
run r 开始运行程序
display disp 跟踪查看某个变量,每次停下来都显示它的值
step s 执行下一条语句,如果该语句为函数调用,则进入函数执行期中的第一条语句
next n 执行下一条语句,如果该语句为函数调用,不会进入函数内部执行(即不会一步步地调试函数内部语句)
print p 打印内部变量值
continue c 继续程序的运行,直到遇到下一个断点
set var name=v 设置变量的值
start st 开始执行程序,在main函数的第一条语句前面停下来
file 装入需要调试的程序
kill k 终止正在调试的程序
watch 监视变量值的变化
backtrace bt 查看函数调用信息(堆栈)
frame f 查看栈帧
quit q 退出GDB环境
使用core dump文件

当编译好的 C 程序在 Linux 中运行异常退出时，会在程序所在目录内生成一个 core dump 文件。

例：gdb ./a core.113，即可快速定位程序异常退出的位置

代码风格

防御式编程

防止程序使用者的部分恶意输入导致程序崩溃

  1. 别让代码附带有任何假设
  2. 没有被写下来的假设会导致各种问题
  3. 特别是代码规模变大时更是如此
检查
  1. 检查所有外来数据的值
  2. 检查所有输入的参数
  3. 决定如何对不符合预期的输入进行处理
  • 常见的检查项
    • 检查确认所有的数组没有越界
    • 解指针引用前确认他们都是非空的
    • 确保函数的参数都是有效的
    • 检查函数的返回值是符合预期的要求的
    • ……
  • 常见的处理方法
    • 使用一个无害的值
    • 使用最近一次的合法值
    • 记录一个警告的日志
    • 过程中抛出一个异常
    • 进入一个专门的异常处理子程序
    • 终止程序运行
    • ……

代码风格基础

命名

  • 符合语义

    变量名 名词或名词化的动词

    函数/方法名 动词+名词/形容词

  • 清晰/避免缩写

    不要有歧义

    与类型对应

  • 同类用途风格保持一致

    大/小驼峰、下划线、中划线命名

注释
  1. 总结代码
    方便读懂代码
  2. 标记一些内容
    e.g. Todo
  3. 代码意图说明
    指出结局的问题
  4. 传达代码无法表述的信息
    版权声明、版本号、保密要求etc.

自注解好于强行注释

格式
  • 缩进
    统一
  • 空格
    1. 双目运算符前后
    2. 同基本的英文写作要求
    3. 括号的先后是否换行要统一
  • 换行
    1. 不同逻辑的连续代码之间可以有一空行
    2. 函数之间可以有一空行
    3. 一些语言大括号之前是否换行要统一
